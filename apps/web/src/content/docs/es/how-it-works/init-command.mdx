## Comando Init

El comando `init` copia una plantilla de Devsync a tu directorio actual, configurando todo lo que necesitas para comenzar a construir tu portafolio.

<Steps>

1. **Verificar bandera de plantilla**: Primero, verifica si la bandera `--template` está proporcionada. Si lo está, el comando divide la cadena por `/` y la limpia.

2. **Clonar el repositorio**: Si la bandera `--template` no está proporcionada, clona la plantilla predeterminada: `https://github.com/jannael/devsync-default-template`.

3. **Limpiar historial de Git**: Elimina el directorio `.git` para asegurar un historial de Git limpio, separado del historial de la plantilla.

4. **Inicializar nuevo repositorio**: Inicializa un nuevo repositorio de Git y hace el primer commit, permitiéndote agregar tu propio remote origin.

</Steps>

<Aside type="tip">Después de ejecutar `init`, sigue las instrucciones en pantalla para crear tu repositorio y subir tus cambios.</Aside>

## Qué Se Crea

Después de ejecutar `init`, tendrás:

- `DEVSYNC.json` - Tu configuración de portafolio
- Archivos fuente de la plantilla - Tu código de portafolio
- Flujo de GitHub Actions - Actualizaciones automáticas en cambios de `DEVSYNC.json`
- Agent skills - Ayudante de IA para operaciones de portafolio

## Siguientes Pasos

1. Crea un repositorio de GitHub con tu nombre de usuario
2. Agrega remote origin: `git remote add origin https://github.com/[username]/[username].git`
3. Sube tus cambios: `git push -u origin main`
4. Rellena `DEVSYNC.json` con tu información
5. Sube para ver tu portafolio actualizarse automáticamente
