## Comando Create Template

El comando `create-template` genera una estructura completa de plantilla de Devsync que puedes personalizar para tu portafolio o distribuir a otros.

<FileTree>

- .agents
  - skills
    - devsync-portfolio
      - SKILL.md
- .github
  - workflows
    - update-on-devsync-change.yml
- public
  - poster-dark-mode.webp
  - poster-light-mode.webp
- src
  - devsync
    - devsync-validator.ts
    - devsync.ts
    - fields-translations.ts
- DEVSYNC.json
- package.json

</FileTree>

### Uso de Archivos

- **`.agents/skills/devsync-portfolio/`**: Agente skill que ayuda a la IA a entender cómo operar la plantilla y construir portafolios desde prompts.
- **`.github/workflows/update-on-devsync-change.yml`**: GitHub Action que actualiza automáticamente tus archivos de portafolio cada vez que `DEVSYNC.json` se modifica.
- **`src/devsync/devsync-validator.ts`**: Validador Zod para el archivo `DEVSYNC.json`.
- **`src/devsync/devsync.ts`**: Wrapper que simplifica usar archivos de Devsync dentro de tu proyecto.
- **`src/devsync/fields-translations.ts`**: Archivo de traducción para contenido localizado.
- **`public/poster-[theme]-mode.webp`**: Imágenes de vista previa usadas para mostrar tu portafolio en el sitio web de Devsync.

<Aside type="tip">
	No necesitas crear todos estos archivos manualmente. Ejecuta `bunx @jannael/devsync create-template` para generar la estructura completa.
</Aside>
