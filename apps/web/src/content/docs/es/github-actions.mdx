## Resumen

Devsync está diseñado para funcionar perfectamente con GitHub Actions. Puedes actualizar automáticamente tu perfil de LinkedIn, README de GitHub, CV y académicos cada vez que cambies `DEVSYNC.json`.

<Aside type="tip">
	El comando `build` está pensado principalmente para CI/CD, pero puedes ejecutarlo localmente si tienes Bun y Puppeteer instalados.
</Aside>

## Ejemplo de Flujo Completo

Aquí está el flujo completo que impulsa `update-on-devsync-change.yml`:

```yaml
name: Update CV, README, LinkedIn & Academics

on:
  push:
    paths:
      - 'DEVSYNC.json'

permissions:
  contents: write

jobs:
  update:
    name: Update artifacts
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v6
        with:
          persist-credentials: true
          fetch-depth: 0

      - name: Setup Bun
        uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'

      - name: Install Puppeteer dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y \
            libx11-6 libxcomposite1 libxcursor1 libxdamage1 libxext6 \
            libxfixes3 libxi6 libxtst6 libnss3 libglib2.0-0 libgtk-3-0t64 \
            libasound2t64 libxrandr2 libpangocairo-1.0-0 libatk1.0-0 \
            libatk-bridge2.0-0 libcups2 libdbus-1-3 libdrm2 libgbm1 \
            libgdk-pixbuf2.0-0

      - name: Install dependencies
        run: bun install

      - name: Update artifacts
        run: bunx @jannael/devsync build

      - name: Push changes
        uses: stefanzweifel/git-auto-commit-action@v7
        with:
          commit_message: 'docs: auto-update artifacts from DEVSYNC.json'
```

## Desglose del Flujo

### 1. Trigger

```yaml
on:
  push:
    paths:
      - 'DEVSYNC.json'
```

Se activa solo cuando `DEVSYNC.json` cambia, evitando ejecuciones innecesarias.

### 2. Checkout

```yaml
- name: Checkout repository
  uses: actions/checkout@v6
  with:
    persist-credentials: true
    fetch-depth: 0
```

- `persist-credentials: true` - Permite subir cambios de vuelta
- `fetch-depth: 0` - Obtiene el historial completo (necesario para algunas operaciones de Git)

### 3. Configurar Runtimes

```yaml
- name: Setup Bun
  uses: oven-sh/setup-bun@v2
  with:
    bun-version: latest

- name: Setup Node.js
  uses: actions/setup-node@v4
  with:
    node-version: '22'
```

Tanto Bun como Node.js son requeridos:

- **Bun** - Ejecuta el CLI y construye el proyecto
- **Node.js** - Requerido por Puppeteer para generación de PDF

### 4. Instalar Dependencias del Sistema

```yaml
- name: Install Puppeteer dependencies
  run: |
    sudo apt-get update
    sudo apt-get install -y \
      libx11-6 libxcomposite1 libxcursor1 libxdamage1 libxext6 \
      libxfixes3 libxi6 libxtst6 libnss3 libglib2.0-0 libgtk-3-0t64 \
      libasound2t64 libxrandr2 libpangocairo-1.0-0 libatk1.0-0 \
      libatk-bridge2.0-0 libcups2 libdbus-1-3 libdrm2 libgbm1 \
      libgdk-pixbuf2.0-0
```

Estas son librerías de Chrome/Chromium necesarias para generación de PDF en modo headless.

<Aside type="caution">
	¡No saltes el paso de dependencias del sistema! Sin ellas, la generación de PDF fallará con errores como "Chrome not found" o "Cannot start Chrome".
</Aside>

### 5. Instalar Dependencias

```yaml
- name: Install dependencies
  run: bun install
```

Instala las dependencias del proyecto incluyendo Puppeteer.

### 6. Ejecutar Build

```yaml
- name: Update artifacts
  run: bunx @jannael/devsync build
```

Este comando:

1. Valida `DEVSYNC.json`
2. Construye el CV para cada idioma
3. Crea el perfil de LinkedIn
4. Crea el perfil de GitHub
5. Crea el perfil académico
6. Genera PDFs

### 7. Subir Cambios

```yaml
- name: Push changes
  uses: stefanzweifel/git-auto-commit-action@v7
  with:
    commit_message: 'docs: auto-update artifacts from DEVSYNC.json'
```

Automáticamente confirma y sube los archivos generados de vuelta al repositorio.

## Ejemplos de Personalización

### Agregar Notificación de Éxito

```yaml
- name: Notify on success
  if: success()
  run: |
    echo "Build completed successfully!"
    # Agrega notificación de Discord/Slack aquí
```

### Caché de Dependencias

Acelera builds cacheando dependencias de Bun:

```yaml
- name: Cache Bun dependencies
  uses: actions/cache@v4
  with:
    path: ~/.bun/install/cache
    key: ${{ runner.os }}-bun-${{ hashFiles('**/bun.lockb') }}
    restore-keys: |
      ${{ runner.os }}-bun-
```

### Múltiples Trabajos (Paralelo)

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: oven-sh/setup-bun@v2
      - run: bun install
      - run: bunx @jannael/devsync build

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to production
        run: echo "Deploy..."
```

## Consideraciones de Seguridad

### Permisos

```yaml
permissions:
  contents: write # Requerido para subir cambios
```

Solo otorga los permisos mínimos necesarios. Para Devsync, `contents: write` es necesario para confirmar archivos generados.

### Secrets

Si tu flujo necesita secrets (ej. tokens de deploy):

```yaml
env:
  DEPLOY_TOKEN: ${{ secrets.DEPLOY_TOKEN }}
```

Almacena secrets en configuración del repositorio → Secrets → Actions.

## Solución de Problemas

### El Flujo No Se Activa

- Verifica que `DEVSYNC.json` esté en el directorio raíz
- Verifica que el patrón de ruta coincida exactamente
- Revisa la pestaña Actions por flujos deshabilitados

### Build Falla

Problemas comunes:

1. **Permiso denegado** - Verifica `permissions: contents: write`
2. **Build falla**: Si usas una plantilla personalizada, asegúrate de que todas las dependencias del sistema estén instaladas en el flujo.

### Generación de PDF Falla

- Verifica que todas las dependencias de Chrome estén instaladas
- Verifica que `node-version: '22'` esté configurado
- Asegúrate de que `fetch-depth: 0` en el paso de checkout
